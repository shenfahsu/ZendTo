Using the RPM

The main problem with RedHat / CentOS / Fedora systems is that they do not support SQLite when used in PHP. So you have to use MySQL instead. To do this, look in the directory /opt/zendto/docs/sql and follow the instructions in there to setup the MySQL database required for ZendTo.

You will also need to patch and rebuild PHP as it currently contains bugs that restrict it to uploads of less than 2 Gbytes. These are easily fixed, and this is documented on this site.

All the files comprising Zendto are installed into /opt/zendto, and the temporary files and dropoffs are stored under /var/zendto. You may want to move /var/zendto onto a separate partition with plenty of free space, depending on how many uploads you are expecting to see. The configuration is all contained in /opt/zendto/config.

The DocumentRoot of the website for ZendTo should be set to /opt/zendto/www.

There are also cron jobs added by creating the file /etc/cron.d/zendto.

Errors about virus scanning?

You need to install ClamAV in the form of a few RPMs. The best place to get these is http://packages.sw.be/clamav/. In that directory "el4" means RedHat Enterprise 4 and CentOS 4, "el5" is RedHat Enterprise 5 and CentOS 5, and the "fc" builds are for various versions of Fedora.

You need to download and install the clamav, clamav-db and clamd RPMs.

You then need to add apache to the clamav group so that clamdscan and clamd can access the files you have uploaded. To do this, run this sequence of commands: 

   usermod -a -G apache clamav
   chkconfig clamd on
   service clamd restart

If you want the full list of packages needed by the RPM for full functionality, then please look in the next "Dependencies" section of this page. That will tell you everything you need to install, whatever version of what distribution you are using. They can be find on either the distribution media for your version of Linux or else on the packages.sw.be site which is a member of the RPMForge sites.

MyZendTo

MyZendTo is another package that provides a simple ZendTo-based filestore service where users can just store groups of files for access later via the web. They can optionally send the dropoffs to other people as well, but primarily they are storing dropoffs for themselves.

To enable MyZendTo, just create another web site on your server whose "DocumentRoot" is set to "/opt/zendto/myzendto.www" and look at that using a web browser.

Dependencies

ZendTo needs quite a few packages to be installed before it can be really useful. I would suggest the following steps: 

yum -y install sendmail sendmail-cf
yum -y install httpd mod_ssl
yum -y install mysql mysql-server
chkconfig mysqld on
chkconfig iptables off
chkconfig ip6tables off
chkconfig httpd on
yum -y install php php-cli php-common php-imap php-ldap php-mysql
yum -y install elinks rpm-build
yum -y install vixie-cron crontabs logrotate yum-cron
yum -y update
yum clean all

elinks http://packages.sw.be/
Download from "clamav" dir: clamav clamav-db clamd
Download from "lua" dir: lua lua-devel
Download from "rrdtool" dir: lua-rrdtool perl-rrdtool rrdtool
yum -y install libdbi lua-devel gettext lua ruby xorg-x11-fonts-Type1
rpm -Uvh *rpm
